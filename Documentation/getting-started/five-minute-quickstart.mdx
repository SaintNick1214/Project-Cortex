# Five-Minute Quickstart

Get your first AI agent with persistent memory running in 5 minutes.

<Callout type="info" title="Before You Begin">
  **Time**: ~5 minutes | **Prerequisites**: Node.js 20+, npm or yarn
</Callout>

<Steps>
  <Step title="Install the CLI (30 seconds)">
    Install the Cortex CLI globally:
    
    <Terminal>npm install -g @cortexmemory/cli</Terminal>
    
    Verify installation:
    
    <Terminal>cortex --version</Terminal>
    
    <Callout type="tip" title="Expected Output">
      You should see version `0.27.4` or higher.
    </Callout>
  </Step>
  
  <Step title="Initialize Your Project (1 minute)">
    Create a new Cortex project:
    
    <Terminal>cortex init my-first-agent</Terminal>
    
    The interactive wizard guides you through:
    
    1. **Project name** - Press Enter to accept default
    2. **Convex setup** - Choose "Create new Convex project" (full features, free tier) or "Use local" (fastest, no account needed)
    3. **Template selection** - Choose "vercel-ai-quickstart" for a full working example
    4. **Graph database** - Recommended! Enables relationship queries and knowledge graphs (Docker for local setup, or any Neo4j-compatible DB)
    
    <Callout 
      type="tip" 
      title="What the wizard creates" 
      collapsible
      defaultOpen={false}
    >
      Project structure, dependencies (`@cortexmemory/sdk`, `convex`), environment config (`.env.local`), Convex backend with Cortex functions deployed, and persistent configuration saved to `~/.cortexrc`.
    </Callout>
  </Step>
  
  <Step title="Start Development Services (30 seconds)">
    Start all services:
    
    <Terminal>cortex start</Terminal>
    
    The CLI uses persistent config with absolute paths, so you can run this from any directory.
    
    <Callout type="tip" title="Expected Output" collapsible>
      ```
      Starting 1 deployment(s)...
      
      my-first-agent
      Project: /path/to/my-first-agent
      URL: http://127.0.0.1:3210 (local) or https://your-app.convex.cloud (cloud)
      
      Convex development server started
      All deployments started
      ```
    </Callout>
    
    **What's running:** Convex backend (database + functions), Next.js dev server (if using vercel-ai-quickstart), your app at http://localhost:3000
  </Step>
  
  <Step title="Store Your First Memory">
    Now let's store and search memories:
    
    <Tabs groupId="method">
      <TabItem value="sdk" label="SDK (TypeScript)">

```typescript
import { Cortex } from "@cortexmemory/sdk";

const cortex = new Cortex({
  convexUrl: process.env.CONVEX_URL!,
});

// Store a memory
await cortex.memory.remember({
  memorySpaceId: "my-first-agent",
  conversationId: "conversation-1",
  userMessage: "I prefer dark mode",
  agentResponse: "Got it!",
  userId: "user-123",
  userName: "User",
});

// Search memories
const results = await cortex.memory.search(
  "my-first-agent",
  "what are the user preferences?"
);

console.log("Found:", results.length, "memories");
// ‚Üí Found: 1 memories
```

      </TabItem>
      <TabItem value="cli" label="CLI">

<Terminal>cortex memory store --space my-first-agent --content "I prefer dark mode"</Terminal>

Search memories:

<Terminal>cortex memory search "preferences" --space my-first-agent</Terminal>

      </TabItem>
    </Tabs>
  </Step>
  
  <Step title="Try Your Agent">
    <Tabs groupId="template">
      <TabItem value="vercel" label="Vercel AI Template">

If you selected the `vercel-ai-quickstart` template:

**1. Open your app** at http://localhost:3000

**2. Chat with your agent:**

- Type: "I prefer dark mode" ‚Üí Agent: "I'll remember that!"
- Type: "What are my preferences?" ‚Üí Agent: "You prefer dark mode"

**3. Memory persists across sessions** - Refresh the page, start a new conversation, and the agent still remembers.

      </TabItem>
      <TabItem value="basic" label="Basic Template">

If you selected the `basic` template:

<Terminal>npm start</Terminal>

<Callout type="tip" title="Expected Output">
  ```
  üß† Cortex Memory SDK - Example
  ================================
  
  üíæ Storing a memory...
  ‚úì Memory stored!
  
  üîç Searching memories...
  ‚úì Found 1 relevant memories
  
  Memory: I prefer dark mode
  ```
</Callout>

      </TabItem>
    </Tabs>
  </Step>
</Steps>

<Callout type="tip" title="Congratulations!">
  You've built an AI agent with persistent memory! Your agent can remember user preferences, search past conversations, and persist data across restarts.
</Callout>

<Callout type="info" title="About Authentication">
  The quickstart template includes basic auth for development. For production, integrate your own auth system (Auth0, Clerk, NextAuth, etc.) - see the [Auth Integration Guide](/getting-started/auth-integration).
</Callout>

## Explore Your Data

### Using the Convex Dashboard

<Terminal>cortex convex dashboard</Terminal>

This opens the correct dashboard for your deployment (local or cloud).

**What you'll see:**

| Table | Description |
|-------|-------------|
| `conversations` | All conversation threads |
| `memories` | Searchable memory index (vector store) |
| `immutable` | Versioned message history |
| `users` | User profiles |
| `memorySpaces` | Memory space registry |

### Using CLI Commands

```bash
# View database statistics
cortex db stats

# List memory spaces
cortex spaces list

# List memories in a space
cortex memory list --space my-first-agent

# Search memories
cortex memory search "preferences" --space my-first-agent
```

## Interactive Development

For the best experience, use the interactive dashboard:

<Terminal>cortex dev</Terminal>

This provides live service status, streaming logs, auto-reload on changes, and keyboard shortcuts (`c` clear, `s` status, `r` restart, `k` kill ports, `q` quit).

## Common Workflows

<Tabs groupId="workflow">
  <TabItem value="dev" label="Development">

```bash
# 1. Start interactive dev mode
cortex dev

# 2. Make code changes (Convex auto-reloads)

# 3. View live logs in dashboard

# 4. Test with CLI commands
cortex memory list --space my-agent

# 5. Quit when done (press 'q')
```

  </TabItem>
<TabItem value="prod" label="Production">

```bash
# 1. Add production deployment
cortex config add-deployment production \
  --url https://prod.convex.cloud \
  --key "prod|..."

# 2. Deploy backend to production
cortex deploy -d production

# 3. Verify deployment
cortex status -d production
cortex db stats -d production
```

</TabItem>
<TabItem value="multi" label="Multiple Environments">

```bash
# Add cloud deployment
cortex config add-deployment cloud \
  --url https://my-app.convex.cloud \
  --key "prod|..."

# View all deployments
cortex config list

# Target specific deployment with -d flag
cortex db stats -d cloud
cortex deploy -d cloud
cortex status -d local
```

</TabItem>
</Tabs>

<Callout type="warning" title="Troubleshooting" collapsible>
  **Connection errors?** Ensure Convex is running (`npx convex dev`) and `CONVEX_URL` is set in `.env.local`.
  
  **Port conflicts?** Run `cortex dev` then press `k` to kill stuck processes.
  
  **Vector search not working?** Vector search requires cloud Convex. Add a cloud deployment with `cortex config add-deployment cloud`, then use `-d cloud` flag.
</Callout>
